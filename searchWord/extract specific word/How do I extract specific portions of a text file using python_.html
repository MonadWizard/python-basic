How do I extract specific portions of a text file using Python?

Updated: 12/29/2017 by Computer Hope

Note: For all of these examples, we'll be using Python version 3.4. Most systems come pre-installed with Python 2.7. While Python 2.7 is used in most legacy code, Python 3 is the present and future of the Python language. Unless you have a specific reason to write or support legacy Python code, we recommend working in Python 3.

The command alias python, on most systems launch Python 2.7. You can check your version by invoking Python with the --version option:

Input:

python --version

Output:

Python 2.7.9

To run python 3.4, the command is python3.4. You can check your version:

Input:

python3.4 --version

Output:

Python 3.4.2

If Python 3.4 is not installed on your system, you can install it using your package manager. For instance, on Debian or Ubuntu, you can install it with the command:

sudo apt-get update && sudo apt-get install python3.4

Running python3.4 with no options will start the interactive interpreter. For more information about using the interpreter, see Python Overview: Using the Python Interpreter. If you accidentally enter the interpreter, you can exit it using the command exit().

Running python3.4 with a file name will interpret that python program. For instance:

python3.4 my-python3-program.py

...runs the program contained in the file my-python3-program.py.

Okay, let's get down to business. How can we use Python to extract text from a text file?

First Things First: Reading Data from a Text File

First, let's read a text file. Let's say we're working with a file, lorem.txt, which contains a few lines of Latin:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc fringilla arcu congue metus aliquam mollis.
Mauris nec maximus purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus eo. Duis id commodo orci.
Quisque at dignissim lacus.

(In all the examples that follow, we work with text contained in this file. Feel free to create it yourself, so that you can run the example code using this file as input.)

In Python, reading a text file can be accomplished with the built-in open() function. For example, below is a Python 3 program that opens lorem.txt for reading in text mode, reads the contents into a string variable named contents, closes the file, and then prints the data.

in_file = open("lorem.txt", "rt") # open file lorem.txt for reading text data
contents = in_file.read()         # read the entire file into a string variable
in_file.close()                   # close the file
print(contents)  		          # print contents

Here, in_file is the name we give to our file object. We named it "in_file" to remind ourselves that it is being used only for input (reading), not output (writing).

The 'rt' parameter in the open() function means "we're opening this file to read text data."

The hash mark ("#") means that everything on the rest of that line is a comment, and is ignored by the Python interpreter.

If you save this program in a file called read.py, you can run it with this command:

python3.4 read.py

And displays the contents of lorem.txt:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc fringilla arcu congue metus aliquam mollis.
Mauris nec maximus purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus leo. Duis id commodo orci.
Quisque at dignissim lacus.

Good Practice: Using "with open"

To create an efficient program, It's important to close your open files as soon as possible: open the file, perform your operation, and close it. Don't leave it open for extended periods of time!

A great way to do this is with the with open... as compound statement is the cleanest way to open a file, operate on it, and close the file, all in one easy-to-read block of code. The file is automatically closed when the code block completes.

Using with open... as, we can rewrite our program to look like this:

with open ('lorem.txt', 'rt') as in_file:  # Open file lorem.txt for reading of text data.
contents = in_file.read()# Read the entire file into a variable named contents.
print(contents) # Print contents.

Note: Indentation is important in Python! Python programs use white space at the beginning of a line to define scope, such as a block of code. It's recommended that you use four spaces per level of indentation, and that you use spaces rather than tabs. In all the following examples, make sure your code is indented exactly as it's presented here.

Save the program as read.py and execute it:

python3.4 read.py

Output:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc fringilla arcu congue metus aliquam mollis.
Mauris nec maximus purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus leo. Duis id commodo orci.
Quisque at dignissim lacus.

Reading Text Files Line-By-Line

In the examples so far, we've been reading in the whole file at once. Reading a full file is no big deal with small files, but generally speaking, it's not a great idea. For one thing, if your file is bigger than the amount of available memory, you'll encounter an error.

In almost every case, it's a better idea to read a text file one line at a time.

In Python, the file object is an iterator. An iterator is a type of Python object which behaves in certain ways when operated on repeatedly. For instance, you can use a for loop to operate on a file object repeatedly, and each time the same operation is performed, you'll receive a different, or "next", result.

For text files, the file object iterates one line of text at a time. It considers one line of text a "unit" of data, so we can use a for...in loop statement to iterate on the data one line at a time:

with open ('lorem.txt', 'rt') as in_file:  # Open file lorem.txt for reading of text data.
for line in in_file: # Store each line in a string variable "line"
print(line) # prints that line

Output:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc fringilla arcu congue metus aliquam mollis.
Mauris nec maximus purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus leo. Duis id commodo orci.
Quisque at dignissim lacus.

Notice that we're getting an extra line break (also called a "newline") after every line. That's because two line breaks are being printed: the first one is the line break at the end of every line of our text file. The second line break happens because, by default, print() adds a linebreak of its own at the end of whatever you've asked it to print.

To fix this redundancy, we can either strip the line breaks from our data, or we can tell print() not to add its own linebreak. We'll describe how to do that in a moment.

First, let's put our lines of text into a variable so we can take a closer look at what's going on.

Reading Data Into A Variable

So far, we've printed the file's data, but we haven't done anything else with it. To do anything really useful, we should store it in a variable.

But what type of variable should we use? We could store the data in one long string, but it would be more useful if we store it in a list, where each element in the list is a single line of text.

In Python, lists are similar to (but not the same as!) an array in C or Java. It contains indexed data. (Unlike an array, however, Python lists can contain elements of varying length and type.)

lines = [] #Declare an empty list named "lines"
with open ('lorem.txt', 'rt') as in_file:  #Open file lorem.txt for reading of text data.
for line in in_file: #For each line of text store in a string variable named "line", and
lines.append(line)  #add that line to our list of lines.
print(lines)        #print the list object.

The output of this program is a little different. Instead of printing the lines themselves, this program prints our list object, which looks like this:

['Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc \n', 'fringilla arcu congue metus aliquam mollis.
Mauris nec maximus \n', 'purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus \n', 'leo. Duis id commodo orci.
Quisque at dignissim lacus. \n']

Here, we see the raw contents of the list. The whole thing is contained in brackets. Its individual elements are enclosed in single-quotes, and delimited with commas. Also, note that each element is a string, and that the line breaks (\n) are preserved at the end of each string.

Much like an array in C or Java, we can access the elements by specifying an index number after the variable name, in brackets. Index numbers start at zero — other words, the nth element of a list has the numeric index n-1.

Note: If you're wondering why the index numbers start at zero instead of one, you're not alone! Computer scientists have debated the usefulness of zero-based numbering systems in the past. In 1982, Edsger Dijkstra gave his opinion on the subject, explaining why zero-based numbering is the best way to index data in computer science. You can read the memo yourself — he makes a compelling argument!

We can print the first element of lines by specifying index number 0, which we indicate with brackets after the list name:

print(lines[0])

Output:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc

Or the fourth line, by specifying index number 3:

print(lines[3])

Output:

purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus

But if we try to access an index for which there is no value, we get an error:

print(lines[4])

Output:

Traceback (most recent call last):
File <filename>, line <linenum>, in <module>
print(lines[4])
IndexError: list index out of range

A list object is an iterator, so to print every element of the list, we can iterate over it with for...in:

lines = []             # Declare an empty list named "lines"
with open ('lorem.txt', 'rt') as in_file:  # Open file lorem.txt for reading of text data.
for line in in_file:   # For each line of text in in_file, where the data is named "line",
lines.append(line)     # add that line to our list of lines.
for element in lines:  # For each element in our list,
print(element)         # print it.

Output:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc fringilla arcu congue metus aliquam mollis.
Mauris nec maximus purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus leo. Duis id commodo orci.
Quisque at dignissim lacus.

But we're still getting extra line breaks! Let's figure out how to fix this.

As we mentioned above, each line of our text file ends in a line break (\n), which is being printed. Also, after printing each line, print() adds a line break of its own by default. However, we can change this by specifying an end parameter in our print() call:

print(element, end='')

The above example tells print() to print an empty string ('') instead of a newline at the end. Now our output looks like this:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc fringilla arcu congue metus aliquam mollis.
Mauris nec maximus purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus leo. Duis id commodo orci.
Quisque at dignissim lacus.

But what if we want to remove the line breaks that exist in our data? We can strip them. String objects have many useful methods for operating on their data, including a method called rstrip(), which strips characters from the end of a string. To automatically remove line breaks from the end of our strings as we read them in, our program would look like this:

lines = []                  # Declare an empty list named "lines"
with open ('lorem.txt', 'rt') as in_file:  # Open file lorem.txt for reading of text data.
for line in in_file:  # For each line of text in in_file, where the data is named "line",
lines.append(line.rstrip('\n'))   # add that line to our list of lines, stripping newlines.
for element in lines:            # For each element in our list,
print(element)              # print it.

So we've got our text stored a variable. Great! Now let's search it.

Searching For Substrings Using A String's find() Method

Let's say we want to locate every instance of a string in our text file. For instance, maybe we need to know where every "e" is. We can accomplish this using the string object's find() method.

Our list object lines stores each line of our text as a string object, and string objects have a method, find(), which locates the first occurrence of a substrings within the string.

For example, we can use the string object's find() method to look for the letter "e" in the first line of our text by using lines[0], which is the first element of the list lines. The lines[0] element is a string object, and has a find() method:

print(lines[0].find("e"))

Output:

3

The above output tells us that the letter e is the fourth character. (Just like the index of an array, string indexes are zero-based, so index 3 is character number four.)

find can accept two additional parameters: integers representing indexes where it should start and stop its search. If we specify only one integer, it will start the search at that string index, and stop at the end of the string. For instance, to search from the fifth character onwards:

print(lines[0].find("e", 4))

Output:

24

So the next "e" is the 25th letter (index 24).

To search only from index 25 through 35:

print(lines[0].find("e", 25, 35)

Output:

32

So between indexes 25 and 35, the first "e" is at index 32.

If find() doesn't find anything, it will return -1:

print(lines[0].find("e", 25, 30))

-1

There were no "e" occurrences between indexes 25 and 30.

But what if we want to locate every occurrence of a substring, not just the first one we encounter? We can manually iterate over the string, repeating our search from the location of the last known occurrence.

In this example, we'll use a while loop to repeatedly find the letter "e". When an occurrence is found, we call find again, starting from a new location in the string. Specifically, the location of the last occurrence, plus the length of the string (so we can move forward past the last one). When find returns -1, or the start index exceeds the length of the string, we stop.

index = 0              # index represents where in the string we begin looking.
str = lines[0]           # str is our string to search. In this case, lines[0].
substr = "e"            # substr is our substring for which to search. In this case, "e".
while index < len(str): #While index is a number smaller than the number of letters in str.
index = str.find(substr, index) #set index to location of first remaining occurrence of "e"
if index == -1:         # If nothing was found,
break            # exit the while loop. Otherwise,
print("Index: ", index)     # print the index where "e" was found, and
index += len(substr)      # increment the index by the number of characters in substr, and repeat.

Output:

Index: 3 Index: 24 Index: 32 Index: 35 Index: 51

Sure enough, the letter "e" occurs five times in the first line of our file:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc ^3 ^24 ^32^36 ^51

Working With Multiple Lines Of Text

Let's perform the same find() procedure, but this time let's do it for every line stored in our list.

We should probably keep track of what line we're searching so that we can make use of that information. An easy way to do this is to use a built-in function called enumerate().

enumerate() takes another sequential object (such as a list), and returns an enumerate object.

If you remember, this is what our raw list object looks like:

print(lines)

Output:

['Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc ' 'fringilla arcu congue metus aliquam mollis.
Mauris nec maximus ' 'purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus ' 'leo. Duis id commodo orci.
Quisque at dignissim lacus. ']

We can create an enumerate object from lines like this:

enumerate(lines)

Enumerate objects don't have a printable form on their own, but we can use the list() function to create a list of our enumerate object, just to see what's going on:

print(list(enumerate(lines)))

Output:

[(0, 'Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc '), (1, 'fringilla arcu congue metus aliquam mollis. Mauris nec maximus '), (2, 'purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus '), (3, 'leo. Duis id commodo orci. Quisque at dignissim lacus. ')]

The enumerate object creates a sequence of tuples from our list. There are four tuples in this list, and each one is grouped inside parentheses.

Each of these tuples comprises two pieces of data: an index number, and the data at that index. Great — How can we use this information?

for...in, which we've already used, can keep track of multiple variables as it iterates. For example, consider this for...in loop:

for linenum, line in enumerate(lines):
print(linenum, line)

Output:

0 Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc 1 fringilla arcu congue metus aliquam mollis.
Mauris nec maximus 2 purus. Maecenas sit amet pretium tellus. Praesent sed rhoncus 3 leo. Duis id commodo orci.
Quisque at dignissim lacus.

Now we've got line numbers! Pretty neat. Here, for...in iterates over our enumerate object, which is created by iterating over lines. What for...in gets back is a series of tuples that looks like this:

(0, 'Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nunc ')

...and it places those values into the variables linenum and line, respectively.

Let's use an enumerate object to spice up our substring search, returning line numbers to accompany our string indexes:

substr = "e"           # Substring to search for, in this case a single character
for linenum, line in enumerate(lines):  # For every line in lines, enumerated by linenum,
index = 0               # Set the search index to the first character,
str = lines[linenum]	# and store the line in a string variable, str.
while index < len(str): # While search index is less than the length of the string:
index = str.find(substr, index)# If substring is located, set search index to that location.
if index == -1:         # If nothing is found,
break                   # break out of the while loop. Otherwise,
print("Line: ", linenum, "Index: ", index) # Print the linenum and index of the located substr.
index += len(substr)    # Before repeating search, increment index by length of substr.

Now we know the location of every letter "e" in our text file:

Line: 0 Index: 3 Line: 0 Index: 24 Line: 0 Index: 32 Line: 0 Index: 35 Line: 0 Index: 51 Line: 1 Index: 20 Line: 1 Index: 23 Line: 1 Index: 52 Line: 2 Index: 9 Line: 2 Index: 11 Line: 2 Index: 22 Line: 2 Index: 27 Line: 2 Index: 34 Line: 2 Index: 44 Line: 2 Index: 46 Line: 2 Index: 51 Line: 3 Index: 1 Line: 3 Index: 33

The above method will work for a substring of any length. But maybe you need to search for something a little more complex. Let's look at how regular expressions can be incorporated in your search.

Incorporating Regular Expressions

You can increase the power of your search using regular expressions, which allows you to search for a wide array of text patterns when processing your file with Python.

The Python regular expressions module is called re. To use it in your program, import the module before you use it:

import re

The re module implements regular expressions by compiling a search pattern into a pattern object. Methods of this object can then be used to perform match operations.

For example, let's say you want to search for any word in your document which starts with the letter d and ends in the letter r. We can accomplish this using the regular expression "\bd\w*r\b". What does this mean?

character sequence meaning
\b A word boundary matches an empty string (anything, including nothing at all), but only if it appears before or after a non-word character. "Word characters" are the digits 0 through 9, the lowercase and uppercase letters, or an underscore ("_").
d Lowercase letter d.
\w* \w represents any word character, and * is a quantifier meaning "zero or more of the previous character." So \w* will match zero or more word characters.
r Lowercase letter r.
\b Word boundary.

So this regular expression will match any string which can be described as "A word boundary, then a lowercase d, then zero or more word characters, then a lowercase r, then a word boundary." strings that can be described this way include the words destroyer, dour, and doctor, and the abbreviation dr.

To use this regular expression in Python search operations, we first compile it into a pattern object. For instance, this Python statement:

pattern = re.compile(r"\bd\w*r\b")

...will create a pattern object named pattern which we can use to perform searches using that regular expression.

Note: The letter r before our string in the statement above is important. It tells Python to interpret our string as a raw string, exactly as we've typed it. If we didn't prefix the string with an r, Python would interpret the escape sequences such as \b in other ways. Whenever you need Python to interpret your strings literally, specify it as a raw string by prefixing it with r.

Now we can use the pattern object's methods, such as search() to search a string for the compiled regular expression, looking for a match. If it finds one, it will return a special result called a match object. Otherwise, it returns None, a built-in Python constant that is used like the boolean value "false".

For instance:

import re # import the regular expressions module
str = "Good morning, doctor." # string to search
pat = re.compile(r"\bd\w*r\b") # compile the regex "\bd\w*r\b" into a pattern object
if pat.search(str) != None: # Search for the pattern. If the result is NOT false,
print("Found it.") # report a positive result.

Output:

Found it.

To perform a case-insensitive search, you can specify the special constant re.IGNORECASE in the compile step:

import re
str = "Hello, DoctoR."             # String has a capital D and R,
pat = re.compile(r"\bd\w*r\b", re.IGNORECASE) # but matches because pattern ignores case
if pat.search(str) != None:
print("Found it.")

Output:

Found it.

For more information about how to create a regular expression to meet your needs, check out our Regular Expressions Quick Reference.

Putting It All Together

So now we can open a file, read the data into a list, enumerate the lines of our file, and locate a substring in any given line. Let's use this to build some example programs.

Print all lines containing substring

The program below reads a log file line by line. If the line contains the word "error", it is added to a list called err_occur. If not, it is discarded (not added to the list), conserving memory. Because the search is case-insensitive, the string is made lowercase by using lower() before performing the search to register a match even if the original text was in uppercase.

Notice that the find() method is called directly on the result of line.lower() and is an example of method chaining. Also, notice that we use try ... except to handle an exception if the log file is not found: if logfile.txt doesn't exist, program execution will skip to the except FileNotFoundError code block.

The program below also uses the sep= parameter in the print() statement, which defines a character or string which will separate each item printed. Here, we set sep to an empty string to output a little neater by telling print() not to put spaces in between each item it prints.

err_occur = []                         # The list where we will store results.
substr = "error"                        # Substring to use for search.
try:                              # Try to:
with open ('logfile.txt', 'rt') as in_file:        # open file for reading text.
for linenum, line in enumerate(in_file):    # Keep track of line numbers
if line.lower().find(substr) != -1: #If case-insensitive substring search matches, then:
err_occur.append((linenum, line.rstrip('\n'))) # strip linebreaks, store line and line number in list as tuple.
for linenum, line in err_occur:              # Iterate over the list of tuples, and
print("Line ", linenum, ": ", line, sep='')  # print results as "Line [linenum]: [line]".
except FileNotFoundError:                   # If log file not found,
print("Log file not found.")                # print an error message.

Example output:

Line 4237: Feb 20 05:24:12 kernel: [ 16.449901] EXT4-fs (sda1): re-mounted. Opts: errors=remount-ro
Line 5399: Feb 20 07:45:59 host org.a11y.Bus[3508]: g_dbus_connection_real_closed: Remote peer error: returned 0 bytes on an async read. Exiting.
Line 5401: Feb 20 07:45:59 org.a11y.Bus[5501]: XIO: fatal IO error 11 (Resource temporarily unavailable) on server

Extract all lines containing substring, using regex

The program below is the same as the above program, but using the re regular expressions module.

import re
err_occur = []                         # The list where we will store results.
pattern = re.compile("error", re.IGNORECASE)  # Compile a case-insensitive regex pattern.
try:                              # Try to:
with open ('logfile.txt', 'rt') as in_file:        # open file for reading text.
for linenum, line in enumerate(in_file):        # Keep track of line numbers.
if pattern.search(line) != None:          # If substring search finds a match,
err_occur.append((linenum, line.rstrip('\n'))) # strip linebreaks, store line and line number in list as tuple.
for linenum, line in err_occur:              # Iterate over the list of tuples, and
print("Line ", linenum, ": ", line, sep='') # print results as "Line [linenum]: [line]".
except FileNotFoundError:                   # If log file not found,
print("Log file not found.")                # print an error message.

Extract all lines containing a phone number

The program below prints any line of a text file, info.txt, which contains a US or international phone number. It accomplishes this with the regular expression "(\+\d{1,2})?[\s.-]?\d{3}[\s.-]?\d{4}". (For more information about regular expressions, see the Regular Expressions Quick Reference guide.)

import re                           # Import the regex module.
err_occur = []                         # The list where we will store results.
pattern = re.compile(r"(\+\d{1,2})?[\s.-]?\d{3}[\s.-]?\d{4}") # Compile regular expression to match any phone number.
try:                              # Try to:
with open ('info.txt', 'rt') as in_file:          # open file for reading text.
for linenum, line in enumerate(in_file):        # Keep track of line numbers.
if pattern.search(line) != None:          # If pattern search finds a match,
err_occur.append((linenum, line.rstrip('\n'))) # strip linebreaks, store line and line number as tuple.
for linenum, line in err_occur:              # Iterate over list of tuples,
print("Line ", linenum, ": ", line, sep='')  # print results as "Line [linenum]: [line]".
except FileNotFoundError:                   # If input file not found,
print("Input file not found.")               # print an error message.

Sample output:

Line 2: My phone number is 731.215.8881. Line 4: You can reach Mr. Walters at (212) 558-3131. Line 7: His lawyer, Mrs. Kennedy, can be reached at +12 (123) 456-7890 Line 10: You can contact his office at (888) 312.8403, extension 12.

Search a dictionary for words

The program below searches the dictionary for any words that start with d and end in r. For input, it uses a dictionary file included on many Unix systems, /usr/share/dict/words.

import re
filename = "/usr/share/dict/words"
pattern = re.compile(r"\bd\w*r\b", re.IGNORECASE)
with open(filename, "rt") as in_file:
for line in in_file:
if pattern.search(line) != None:
print(line, end='')

Output:

daalder dabber dabbler dabster dacker ... dyker dynameter dynamiter dynamometer dynamotor
